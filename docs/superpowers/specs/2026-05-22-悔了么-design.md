# 悔了么 — 设计规格

## 概述

一个单页面 Web 工具，帮助用户计算"如果当时买了某只股票，现在能赚多少钱"，配合调侃文案增强情绪共鸣。响应式设计，手机端和电脑端均可使用。

---

## 场景定义

**唯一场景：后悔没买。**

用户填入"当时可以买入的价格"和"现在的价格"，工具计算错失的潜在收益。不涉及"买了之后跌了"的亏损场景。

---

## 技术方案

**纯 HTML 单文件（`index.html`）**，内联 CSS 和 JavaScript，零依赖，无构建步骤。

- 可直接双击打开，也可部署到任何静态托管（GitHub Pages、Vercel 等）
- 响应式布局：移动端全宽，桌面端居中（最大宽度 480px）

---

## 视觉风格

- **背景**：纯黑（`#0a0a0a`）
- **文字**：纯白（`#ffffff`），层级用深灰（`#555`、`#333`）区分
- **卡片**：深灰背景（`#141414`），细边框（`#222`）
- **主按钮**：白底黑字，唯一高对比元素
- **字体**：系统字体栈（PingFang SC / Helvetica Neue / sans-serif）
- **风格**：极简，无多余颜色，无渐变，无图标装饰

---

## 输入字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 股票名称 | 文本 | 否 | 仅用于展示，不做验证 |
| 当时价格 | 数字 | 是 | 单位：元，支持小数 |
| 现在价格 | 数字 | 是 | 单位：元，支持小数 |
| 打算买多少股 | 数字 | 二选一 | 填写后自动换算投入金额 |
| 或投入多少钱 | 数字 | 二选一 | 填写后自动换算股数 |

**二选一逻辑**：股数和金额两个输入框同时显示，用户填任意一个，另一个实时自动换算（基于"当时价格"计算）。两个都填时，以最后编辑的那个为准。

---

## 计算逻辑

```
股数 = 投入金额 ÷ 当时价格（如用户填金额）
投入金额 = 股数 × 当时价格（如用户填股数）

当前市值 = 股数 × 现在价格
错失金额 = 当前市值 - 投入金额
涨幅     = (现在价格 - 当时价格) ÷ 当时价格 × 100%
```

---

## 结果展示

点击"算一算"后，结果卡片出现（或更新），包含以下内容：

### 1. 价格对比行
横排显示：当时价格 → 现在价格 → 涨幅百分比

### 2. 错失金额（大字主体）
```
你错失了
¥XXX,XXX
```
数字加千分位格式化，保留两位小数。

### 3. 调侃换算文案
固定换算三项（价格为固定常量，不联网）：
- 瑞幸咖啡：¥30/杯
- iPhone 16 Pro Max：¥9,999/台
- 北京平均月薪：¥12,000/月

文案格式：
```
相当于 X 杯瑞幸咖啡
或 X 台 iPhone 16 Pro Max
或 X 个月北京平均工资
```
数值用 `Math.floor` 取整，不显示小数。

### 4. 底部吐槽语
随机从以下文案中选一条展示：
- 「当时只差一个勇气」
- 「纸上谈兵，白纸一张」
- 「你的钱，在别人口袋里」
- 「错过的不是股票，是自由」
- 「下次一定」

---

## 交互细节

- **实时换算**：用户在股数/金额输入框输入时，另一个输入框实时更新
- **输入校验**：点击"算一算"时检查"当时价格"和"现在价格"是否已填，以及股数/金额至少填一个；不满足时输入框高亮提示
- **结果动画**：结果卡片用 CSS fade-in 淡入显示
- **数字格式**：所有金额显示千分位分隔符，保留两位小数

---

## 页面结构

```
<body>
  <main>
    <header>          <!-- 标题 + 副标题 -->
    <section.input>   <!-- 输入卡片 -->
    <section.result>  <!-- 结果卡片（默认隐藏，计算后显示）-->
  </main>
</body>
```
